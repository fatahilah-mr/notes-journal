3. Fitur Utama (Sesuai PRD + Fase 2)

  • Header info pusat – nama kota, suhu Celsius, deskripsi, kelembapan & angin.
  • Pencarian tersembunyi – ikon kaca pembesar dengan animasi slide-down, input di-trim otomatis, error handling jelas.
  • Hourly forecast (24 jam / 3 jam) – horizontal scroll tanpa scrollbar, menampilkan jam, ikon cuaca (fallback jika gagal load), dan suhu.
  • 5-day forecast – filter data jam terdekat 12:00 (metode A), label “Hari ini” / “Besok” / nama hari, daftar vertikal di mobile dan grid 2 kolom di desktop.
  • Background dinamis – gradien berubah otomatis sesuai kondisi cuaca (Clear, Clouds, Rain, dll) dengan transisi halus.
  • GPS (Geolocation) – deteksi lokasi pengguna dan tampilkan cuaca setempat (nice-to-have terintegrasi penuh).
  • Loading & error state – spinner CSS murni, pesan error informatif, tombol retry.
  • Responsif mobile-first – media queries untuk tablet dan desktop (layout daily menjadi grid 2 kolom, ukuran card membesar).
  • Zero library & zero framework – semua kode ditulis manual, fetch API, Promise.all, manipulasi DOM murni.

4. Alur Kerja Pengembangan (Step-by-Step sesuai PLAN)

  1. Step 1 – HTML semantik: buat struktur dengan header, main#current-weather, section#hourly-forecast, section#daily-forecast, loading/error overlay, tombol GPS.
  2. Step 2 – CSS glassmorphism: variabel CSS, efek blur, horizontal scroll tanpa scrollbar, spinner, utility .hidden, background gradien sementara.
  3. Step 3 – JavaScript API & fetch: getWeatherData(city) dengan Promise.all untuk /weather dan /forecast, handling error, loading state.
  4. Step 4 – Konversi waktu & filter: fungsi formatTime(), formatDay(), formatDayLabel(), filterDailyForecast (metode A: ambil jam terdekat 12:00), filterHourlyForecast (8 data pertama).
  5. Step 5 – DOM rendering: updateCurrentWeather(), updateHourly(), updateDaily(), renderWeatherData(), fallback ikon .png dengan onerror.
  6. Step 6 – Interaktivitas & final: event listener toggle search, submit pencarian dengan trim, retry, GPS, load data awal “Jakarta”. Plus tambahan dynamic background dan desktop responsif (grid daily, ukuran card).

5. Teknis Pengambilan Data & Filtering

  • API: OpenWeatherMap (gratis) – endpoint /weather (current) dan /forecast (5 hari, per 3 jam).
  • Konversi UNIX: new Date(unix * 1000) untuk waktu lokal.
  • Filter 5 hari: kelompokkan berdasarkan dateKey (YYYY-MM-DD), pilih data dengan |hour - 12| terkecil, urutkan, batasi 5.
  • Filter hourly: ambil 8 data pertama (24 jam ke depan).
  • Ikon: URL https://openweathermap.org/img/wn/${iconCode}.png + atribut onerror ke ikon default 01d.png.

6. Desain Visual (CSS)

Menerapkan glassmorphism dengan backdrop-filter: blur(), border semi-transparan, dan box-shadow. Background body diubah dinamis oleh JavaScript berdasarkan weather[0].main. Efek transisi background 0.5s ease. Di desktop (@media (min-width: 1024px)), #daily-list berubah menjadi grid-template-columns: repeat(2, 1fr) dan ukuran card hourly membesar untuk kenyamanan layar lebar.

7. Dynamic Background (Fase 2 – Terintegrasi Penuh)

Fungsi updateDynamicBackground(weatherMain) memetakan kondisi cuaca seperti Clear, Clouds, Rain, Thunderstorm, Snow, dll ke gradien CSS. Dipanggil dalam renderWeatherData() sehingga setiap kali kota berubah (pencarian, GPS, atau load awal), background menyesuaikan secara otomatis.

8. Deployment ke Vercel

  • Repository dihubungkan ke Vercel (GitHub).
  • Build Command: # tidak ada build – cukup static files (tidak perlu npm run build karena proyek murni statis).
  • Output Directory: .
  • Setiap git push ke branch utama akan memicu redeploy otomatis.
  • Pastikan environment variable API_KEY tidak disimpan di Vercel (karena frontend statis, API key tetap terlihat di browser – ini aman untuk proyek belajar karena OpenWeatherMap gratis dan key dibatasi). Alternatif untuk produksi: gunakan backend proxy, tapi di luar scope.

9. Hal yang Dipelajari & Kendala yang Diatasi

  • Penyelarasan nama file JS – awalnya app.js vs script.js menyebabkan aplikasi tidak berjalan. Diperbaiki dengan konsistensi referensi di index.html.
  • Ikon gagal load – karena ekstensi @2x.png kadang lambat; diganti dengan .png dan menambahkan onerror fallback.
  • Background tidak dinamis – karena fitur belum diimplementasikan. Ditambahkan fungsi mapping dan dipanggil di renderWeatherData.
  • Input spasi berlebih – diatasi dengan .trim() baik di handleCitySearch maupun getWeatherData.
  • Horizontal scroll di mobile – menggunakan overflow-x: auto dan menyembunyikan scrollbar dengan pseudo-element.
  • Paginasi tidak diperlukan – karena data terbatas (8 hourly + 5 daily), namun struktur siap untuk pengembangan lebih lanjut.
  • GPS meminta izin dan gagal di HTTP (localhost) – berfungsi baik di lingkungan HTTPS (Vercel) atau localhost dengan izin.
