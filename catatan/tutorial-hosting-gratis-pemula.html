📌 Apa Itu Hosting?

Hosting adalah tempat menyimpan file website (HTML, CSS, gambar) agar bisa diakses oleh orang lain lewat internet. Analoginya: hosting itu seperti tanah tempat rumah berdiri, sedangkan website adalah rumahnya. Tanpa hosting, website hanya bisa dilihat di komputer Anda sendiri.

Untungnya, ada banyak layanan yang menyediakan hosting gratis untuk website statis (website yang hanya terdiri dari file HTML/CSS/JS, tanpa database atau server rumit). Berikut panduan memilih dan menggunakannya.

💡 Pertama: Siapkan File Website Anda

Sebelum memilih hosting, pastikan file website Anda sudah siap. Untuk website statis, minimal Anda punya:

  • index.html (halaman utama)
  • style.css (gaya visual)
  • script.js (logika interaktif)

File-file ini bisa disimpan dalam satu folder. Jika Anda menggunakan build script (seperti npm run build), hasil build-nya biasanya ada di folder dist/ atau build/.

🏆 5 Pilihan Hosting Gratis Terbaik (Untuk Pemula)

1. Vercel ⚡

Cocok untuk: Pemula yang ingin auto-deploy dari GitHub dan butuh preview link untuk setiap perubahan.

Kelebihan:

  • Proses deploy sangat cepat (tinggal klik, selesai).
  • Dapat preview link otomatis untuk setiap commit.
  • Support berbagai framework (Next.js, React, Vue, dll).

Langkah (sederhana):

  1. Buat akun di vercel.com (pakai akun GitHub).
  2. Klik Add New → Project, pilih repository GitHub yang berisi kode website Anda.
  3. Vercel akan otomatis mendeteksi framework. Klik Deploy.
  4. Setelah selesai, Anda akan mendapat URL seperti namaproject.vercel.app.

2. Netlify 🚀

Cocok untuk: Pemula yang ingin deploy dengan drag-and-drop (tanpa GitHub) atau dari folder.

Kelebihan:

  • Bisa upload folder website langsung dari browser (seret folder ke halaman Netlify).
  • Auto-deploy dari GitHub juga tersedia.
  • Ada form handling gratis (untuk website kontak).

Langkah (drag-and-drop):

  1. Login ke netlify.com.
  2. Di dashboard, seret folder website Anda ke area yang bertuliskan Drag your site folder here.
  3. Tunggu beberapa detik, website langsung live di namaproject.netlify.app.

3. GitHub Pages 🐙

Cocok untuk: Pemula yang sudah punya akun GitHub dan ingin hosting tanpa build script.

Kelebihan:

  • Gratis tanpa batasan bandwidth.
  • Mendukung custom domain gratis.

Langkah:

  1. Buat repository dengan nama username.github.io (ganti username dengan nama GitHub Anda).
  2. Upload semua file website ke repository tersebut.
  3. Buka Settings → Pages, pilih branch main, lalu klik Save.
  4. Website akan live di https://username.github.io.

4. Cloudflare Pages ☁️

Cocok untuk: Pemula yang menginginkan kecepatan akses tinggi (karena CDN Cloudflare).

Kelebihan:

  • Mendukung build otomatis dari Git.
  • SSL gratis dan custom domain.

Langkah:

  1. Login ke pages.cloudflare.com.
  2. Klik Create a project → Connect to Git.
  3. Pilih repository, atur Build Command (jika ada) dan Output Directory.
  4. Klik Save and Deploy.

5. Render 🖥️

Cocok untuk: Pemula yang nantinya ingin upgrade ke hosting dengan backend (Node.js, Python).

Kelebihan:

  • Support static site dan server-side.
  • Auto-deploy dari Git.

Langkah:

  1. Login ke render.com.
  2. Klik New + → Static Site.
  3. Hubungkan repository GitHub, atur Build Command dan Publish Directory.
  4. Klik Create Static Site.

🧭 Rekomendasi Untuk Pemula

  • Jika Anda belum punya akun GitHub: Mulai dengan Netlify (drag-and-drop) – paling mudah.
  • Jika sudah punya GitHub dan ingin otomatis update: Pilih Vercel atau Netlify (auto-deploy).
  • Jika website Anda sangat sederhana (tanpa build): GitHub Pages adalah pilihan paling stabil dan gratis.

🛠️ Istilah Penting (Untuk Pemula)

  • Deploy / Deployment: Proses mengunggah file website ke hosting agar bisa diakses publik.
  • Build Script: Perintah (misal npm run build) yang mengubah kode sumber menjadi file siap hosting. Jika website Anda hanya HTML/CSS/JS murni, Anda tidak perlu build script.
  • Custom Domain: Mengganti URL bawaan (misal namaproject.vercel.app) menjadi domain milik sendiri (misal fatahilah.com).
  • SSL: Pengamanan website agar koneksi aman (ditandai dengan gembok di browser). Semua platform di atas memberikan SSL gratis.

📌 Catatan Penting

  • Semua platform di atas gratis untuk penggunaan pribadi dan proyek kecil.
  • Jika website Anda banyak pengunjung (ribuan per hari), mungkin perlu upgrade ke paket berbayar.
  • Selalu baca dokumentasi resmi masing-masing platform untuk informasi terbaru.
