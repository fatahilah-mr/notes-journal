📌 Deskripsi Proyek

Membangun jaringan gateway fungsional pada RouterBoard MikroTik lewat Winbox. Konfigurasi mencakup setup IPv4 address, aktivasi dan manajemen WLAN interface untuk akses wireless, implementasi Firewall NAT agar jaringan lokal terhubung ke internet, konfigurasi DNS dan DHCP Server, serta pengujian stabilitas koneksi via terminal.

💻 Parameter Konfigurasi Inti (WinBox & CLI)

1. Firewall NAT (Network Address Translation)

Agar seluruh segmen IP lokal (192.168.10.0/26 dan 192.168.20.0/25) bisa terhubung ke internet:

  • Chain: srcnat
  • Out. Interface: ether1-WAN
  • Action: masquerade

2. Layanan DHCP Server & Pool

  • dhcp1: interface ether2-LAN, pool dhcp_pool0, Lease Time 00:10:00.
  • dhcp2: interface wlan1-WLAN, pool dhcp_pool1, Lease Time 00:10:00.

🧪 Dokumentasi Hasil Pengujian (Verification)

1. Pengujian Konektivitas Internet dari Router (Ping 8.8.8.8)

[fatah@Mk Fatah] > ping 8.8.8.8
sent=10 received=10 packet-loss=0%

2. Klien Terhubung ke Wi-Fi tapi Sering Terputus (RTO Tinggi)

  • Kendala: Perangkat klien di wlan1-WLAN tidak stabil, RTT tinggi hingga timeout.
  • Analisis: Signal Strength lemah, interferensi frekuensi karena kanal padat.
  • Solusi: Ganti frekuensi dari auto ke kanal lebih sepi setelah wireless scan, optimasi band ke 2GHz-only-N, matikan protokol wireless tidak perlu.

3. Klien LAN Tidak Mendapatkan IP Otomatis dari DHCP Server

  • Kendala: Klien di ether2-LAN mendapat IP APIPA (169.254.x.x).
  • Analisis: Status dhcp1 invalid karena interface ether2-LAN salah dimasukkan ke bridge.
  • Solusi: Keluarkan ether2-LAN dari bridge, arahkan DHCP Server langsung ke port fisik. Setelah diperbaiki, klien mendapat IP 192.168.10.0/26.
