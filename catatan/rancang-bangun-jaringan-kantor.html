📌 Deskripsi Proyek

Simulasi arsitektur jaringan skala besar menggunakan Cisco Packet Tracer dengan segmentasi inter-VLAN routing pada multilayer switch. Infrastruktur inti dikonfigurasi menggunakan protokol routing dinamis Multi-Area OSPF untuk menyediakan jalur komunikasi redundan, efisien, dan fault tolerance. Proyek ini juga mengintegrasikan distribusi IP dinamis terpusat serta kebijakan keamanan lalu lintas data antar-departemen.

🛠️ Ringkasan Teknologi & Fitur

  • Perangkat lunak: Cisco Packet Tracer
  • Routing dinamis: Multi-Area OSPF (Area 0 & Area 1)
  • Segmentasi jaringan: Inter-VLAN Routing (VLAN 10, 20, 30) pada Multilayer Switch (MLS)
  • Layanan IP: DHCP Server terpusat
  • Keamanan jaringan: Standard & Extended Access Control List (ACL)

💻 Langkah Konfigurasi Utama (Snippet)

1. Konfigurasi Inter-VLAN & DHCP Relay pada Multilayer Switch

vlan 10
 name Lantai-1
vlan 20
 name Lantai-2
vlan 30
 name Lantai-3
exit

ip routing

interface vlan 10
 ip address 192.168.10.1 255.255.255.0
interface vlan 20
 ip address 192.168.20.1 255.255.255.0
interface vlan 30
 ip address 192.168.30.1 255.255.255.0
end

2. Aktivasi Protokol Routing Dinamis Multi-Area OSPF

router ospf 1
 router-id 1.1.1.1
 network 192.168.10.0 0.0.0.255 area 0
 network 192.168.20.0 0.0.0.255 area 0
 network 10.10.10.0 0.0.0.3 area 0
 network 192.168.30.0 0.0.0.255 area 1
exit

🧪 Hasil Pengujian (Verification)

  1. Ping Test: Konektivitas antar-VLAN sukses (Success rate 100%).
  2. OSPF Neighbor: show ip ospf neighbor menunjukkan status FULL/DR dan FULL/BDR.
  3. Routing Table: show ip route menampilkan kode O (OSPF) dan O IA (Inter-Area).

🔍 Catatan Troubleshooting & Solusi

1. Distribusi IP DHCP pada Klien di Lantai 3

  • Kendala: Klien VLAN 30 tidak mendapat IP dari DHCP server (APIPA).
  • Analisis: DHCP request bersifat broadcast, tidak melewati batas subnet/VLAN.
  • Solusi: Konfigurasi DHCP Relay Agent pada MLS di interface VLAN 30 dengan perintah ip helper-address [IP_DHCP_Server].

2. OSPF Neighbor Status Stuck di EXSTART/EXCHANGE

  • Kendala: Adjacency OSPF tidak mencapai FULL.
  • Analisis: Ketidakcocokan MTU interface antara Router Utama dan Router B.
  • Solusi: Samakan nilai MTU atau tambahkan ip ospf mtu-ignore pada interface yang bermasalah.
