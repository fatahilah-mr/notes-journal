Metode Monorepo (Satu Repositori untuk Banyak Web)

  • Masalah: Membuat satu repositori GitHub baru untuk setiap satu file latihan kecil sangat tidak efisien dan mengotori akun GitHub jika bekerja via HP.
  • Solusi Monorepo: Membuat satu repositori besar di GitHub (contoh: learn-to-make-a-website) yang di dalamnya berisi banyak sub-folder proyek mandiri.
  • Pengaturan Root Directory di Vercel: Saat melakukan import repositori yang sama ke Vercel untuk proyek baru, kita cukup mengubah target Root Directory ke nama sub-folder yang spesifik (misal: 01-dashboard-it). Vercel akan otomatis membaca file index.html di dalam folder tersebut dan memberikan URL unik yang berbeda.
